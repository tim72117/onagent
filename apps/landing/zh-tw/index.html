Why onagent

把 AI 接進介面，該有的都幫你準備好

從 tool 定義到型別安全的前端程式碼與安全隔離，一次到位。

一次定義，兩邊產出

一份 tool 定義，自動生成 LLM 用的 JSON schema 與型別安全的 TypeScript ToolHandlers 介面——schema 與前端型別永不漂移。

AI 直接操作你的介面

tool call 會派發到你註冊的 handler：填表單、跳頁、勾選、點按鈕——不只是回一段文字，而是真的動起來。

逐 App 隔離與白名單

每把 API key 由伺服器端解析對應的 app（前端無法偽造），並綁定單一 Origin，未設定時預設 fail-closed 拒絕連線。

Ship in 3 steps

三步驟，建立一個 AI 服務

定義能力、推送到平台、嵌入前端——點左側步驟，看每一步實際的程式碼。

↳ 完成後，AI 收到 prompt 時會挑選對應的 tool，SDK 幫你呼叫 handler、把結果回傳給模型。

tools
appId: demo-app
tools:
  - name: fill_search_form
    description: 填入頁面上的搜尋框並送出。
    parameters:
      type: object
      required: [query]
      properties:
        query:
          type: string
          description: 要搜尋的關鍵字。
# 登入（會開瀏覽器授權，網址只帶一個一次性代碼）
$ onagent login --web

# 把 tool 定義推上去
$ onagent save-tools demo-app tools
✓ Saved 1 tool to "demo-app"

# 隨時列出你的 app
$ onagent list-apps
demo-app        1 tool, has key
import { AgentBridge } from "@onagent/bridge";

const bridge = new AgentBridge({
  url: "wss://your-backend/ws",
  appId: "demo-app",
  apiKey: "YOUR_APP_API_KEY",
  tools: {
    fill_search_form: ({ query }) => setQuery(query),
  },
});

bridge.prompt("搜尋紅色運動鞋");     // 讓 AI 動起來
